# üîó Inheritance

## üìñ Overview

**Inheritance** is a mechanism where a new class (child/subclass) acquires properties and behaviors of an existing class (parent/superclass).

**Key Concepts:**
- Code reusability
- IS-A relationship
- Parent class (superclass, base class)
- Child class (subclass, derived class)
- extends keyword
- Method overriding
- super keyword

---

## üéØ Types of Inheritance in Java

```
1. Single Inheritance
   Parent ‚Üí Child
   
2. Multilevel Inheritance
   Grandparent ‚Üí Parent ‚Üí Child
   
3. Hierarchical Inheritance
   Parent ‚Üí Child1, Child2, Child3
   
4. Multiple Inheritance (NOT supported with classes)
   Parent1, Parent2 ‚Üí Child (Use interfaces instead)
   
5. Hybrid Inheritance (NOT supported with classes)
   Combination of above
```

## üíª Example 1: Single Inheritance

```java
// Parent class
class Animal {
    String name;
    
    void eat() {
        System.out.println(name + " is eating");
    }
    
    void sleep() {
        System.out.println(name + " is sleeping");
    }
}

// Child class inherits from Animal
class Dog extends Animal {
    String breed;
    
    void bark() {
        System.out.println(name + " is barking");
    }
    
    void displayInfo() {
        System.out.println("Name: " + name);
        System.out.println("Breed: " + breed);
    }
}

public class SingleInheritanceDemo {
    public static void main(String[] args) {
        Dog dog = new Dog();
        dog.name = "Buddy";
        dog.breed = "Golden Retriever";
        
        // Inherited methods from Animal
        dog.eat();
        dog.sleep();
        
        // Dog's own methods
        dog.bark();
        dog.displayInfo();
    }
}
```

**Output:**
```
Buddy is eating
Buddy is sleeping
Buddy is barking
Name: Buddy
Breed: Golden Retriever
```

## üíª Example 2: Multilevel Inheritance

```java
// Grandparent class
class Vehicle {
    String brand;
    
    void start() {
        System.out.println("Vehicle started");
    }
}

// Parent class
class Car extends Vehicle {
    int wheels = 4;
    
    void drive() {
        System.out.println("Car is driving");
    }
}

// Child class
class ElectricCar extends Car {
    int batteryCapacity;
    
    void charge() {
        System.out.println("Electric car is charging");
    }
    
    void displayInfo() {
        System.out.println("Brand: " + brand);
        System.out.println("Wheels: " + wheels);
        System.out.println("Battery: " + batteryCapacity + " kWh");
    }
}

public class MultilevelInheritanceDemo {
    public static void main(String[] args) {
        ElectricCar tesla = new ElectricCar();
        tesla.brand = "Tesla";
        tesla.batteryCapacity = 75;
        
        // Methods from Vehicle (grandparent)
        tesla.start();
        
        // Methods from Car (parent)
        tesla.drive();
        
        // ElectricCar's own methods
        tesla.charge();
        tesla.displayInfo();
    }
}
```

## üíª Example 3: Hierarchical Inheritance

```java
// Parent class
class Shape {
    String color;
    
    void displayColor() {
        System.out.println("Color: " + color);
    }
}

// Child class 1
class Circle extends Shape {
    double radius;
    
    double calculateArea() {
        return 3.14 * radius * radius;
    }
}

// Child class 2
class Rectangle extends Shape {
    double length;
    double width;
    
    double calculateArea() {
        return length * width;
    }
}

// Child class 3
class Triangle extends Shape {
    double base;
    double height;
    
    double calculateArea() {
        return 0.5 * base * height;
    }
}

public class HierarchicalInheritanceDemo {
    public static void main(String[] args) {
        Circle circle = new Circle();
        circle.color = "Red";
        circle.radius = 5;
        circle.displayColor();
        System.out.println("Area: " + circle.calculateArea());
        System.out.println();
        
        Rectangle rect = new Rectangle();
        rect.color = "Blue";
        rect.length = 10;
        rect.width = 5;
        rect.displayColor();
        System.out.println("Area: " + rect.calculateArea());
        System.out.println();
        
        Triangle triangle = new Triangle();
        triangle.color = "Green";
        triangle.base = 8;
        triangle.height = 6;
        triangle.displayColor();
        System.out.println("Area: " + triangle.calculateArea());
    }
}
```

## üíª Example 4: Method Overriding

```java
class Parent {
    void display() {
        System.out.println("Parent's display method");
    }
    
    void show() {
        System.out.println("Parent's show method");
    }
}

class Child extends Parent {
    // Override parent's display method
    @Override
    void display() {
        System.out.println("Child's display method");
    }
    
    // show() is inherited as-is
}

public class MethodOverridingDemo {
    public static void main(String[] args) {
        Parent parent = new Parent();
        parent.display();  // Parent's version
        parent.show();
        
        System.out.println();
        
        Child child = new Child();
        child.display();  // Child's version (overridden)
        child.show();     // Inherited from parent
        
        System.out.println();
        
        // Polymorphism
        Parent ref = new Child();
        ref.display();  // Child's version (runtime polymorphism)
        ref.show();
    }
}
```

## üíª Example 5: super Keyword

```java
class Employee {
    String name;
    double salary;
    
    Employee(String name, double salary) {
        this.name = name;
        this.salary = salary;
    }
    
    void displayInfo() {
        System.out.println("Name: " + name);
        System.out.println("Salary: $" + salary);
    }
}

class Manager extends Employee {
    String department;
    double bonus;
    
    Manager(String name, double salary, String department, double bonus) {
        super(name, salary);  // Call parent constructor
        this.department = department;
        this.bonus = bonus;
    }
    
    @Override
    void displayInfo() {
        super.displayInfo();  // Call parent method
        System.out.println("Department: " + department);
        System.out.println("Bonus: $" + bonus);
        System.out.println("Total: $" + (salary + bonus));
    }
}

public class SuperKeywordDemo {
    public static void main(String[] args) {
        Manager manager = new Manager("Alice", 50000, "IT", 10000);
        manager.displayInfo();
    }
}
```

**Output:**
```
Name: Alice
Salary: $50000.0
Department: IT
Bonus: $10000.0
Total: $60000.0
```

## üíª Example 6: Constructor Chain in Inheritance

```java
class Grandparent {
    Grandparent() {
        System.out.println("Grandparent constructor");
    }
}

class Parent extends Grandparent {
    Parent() {
        super();  // Implicit if not written
        System.out.println("Parent constructor");
    }
}

class Child extends Parent {
    Child() {
        super();  // Implicit if not written
        System.out.println("Child constructor");
    }
}

public class ConstructorChainDemo {
    public static void main(String[] args) {
        System.out.println("Creating Child object:");
        Child child = new Child();
        
        System.out.println("\nNote: Parent constructors are called first (top to bottom)");
    }
}
```

**Output:**
```
Creating Child object:
Grandparent constructor
Parent constructor
Child constructor
```

## üíª Example 7: Access Modifiers in Inheritance

```java
class Base {
    public int publicVar = 10;
    protected int protectedVar = 20;
    int defaultVar = 30;  // package-private
    private int privateVar = 40;
    
    private void privateMethod() {
        System.out.println("Private method");
    }
    
    protected void protectedMethod() {
        System.out.println("Protected method");
    }
    
    public void publicMethod() {
        System.out.println("Public method");
    }
}

class Derived extends Base {
    void accessMembers() {
        System.out.println("Public: " + publicVar);  // OK
        System.out.println("Protected: " + protectedVar);  // OK
        System.out.println("Default: " + defaultVar);  // OK (same package)
        // System.out.println(privateVar);  // ERROR! Private not inherited
        
        publicMethod();  // OK
        protectedMethod();  // OK
        // privateMethod();  // ERROR! Private not accessible
    }
}

public class AccessModifiersDemo {
    public static void main(String[] args) {
        Derived obj = new Derived();
        obj.accessMembers();
    }
}
```

## üíª Example 8: Real-World Example - Bank Accounts

```java
class BankAccount {
    protected String accountNumber;
    protected String holderName;
    protected double balance;
    
    BankAccount(String accountNumber, String holderName, double balance) {
        this.accountNumber = accountNumber;
        this.holderName = holderName;
        this.balance = balance;
    }
    
    void deposit(double amount) {
        balance += amount;
        System.out.println("Deposited: $" + amount);
    }
    
    void displayBalance() {
        System.out.println("Balance: $" + balance);
    }
}

class SavingsAccount extends BankAccount {
    private double interestRate;
    
    SavingsAccount(String accountNumber, String holderName, double balance, double interestRate) {
        super(accountNumber, holderName, balance);
        this.interestRate = interestRate;
    }
    
    void addInterest() {
        double interest = balance * interestRate / 100;
        balance += interest;
        System.out.println("Interest added: $" + interest);
    }
}

class CurrentAccount extends BankAccount {
    private double overdraftLimit;
    
    CurrentAccount(String accountNumber, String holderName, double balance, double overdraftLimit) {
        super(accountNumber, holderName, balance);
        this.overdraftLimit = overdraftLimit;
    }
    
    void withdraw(double amount) {
        if (balance + overdraftLimit >= amount) {
            balance -= amount;
            System.out.println("Withdrawn: $" + amount);
        } else {
            System.out.println("Insufficient funds!");
        }
    }
}

public class BankAccountDemo {
    public static void main(String[] args) {
        System.out.println("=== Savings Account ===");
        SavingsAccount savings = new SavingsAccount("SAV001", "Alice", 1000, 5);
        savings.displayBalance();
        savings.deposit(500);
        savings.addInterest();
        savings.displayBalance();
        
        System.out.println("\n=== Current Account ===");
        CurrentAccount current = new CurrentAccount("CUR001", "Bob", 500, 1000);
        current.displayBalance();
        current.withdraw(1200);
        current.displayBalance();
    }
}
```

## üìä Inheritance Key Points

| Concept | Description |
|---------|-------------|
| **extends** | Keyword to inherit from class |
| **super** | Reference to parent class |
| **@Override** | Annotation for overriding methods |
| **IS-A** | Relationship (Dog IS-A Animal) |
| **Code Reuse** | Inherit existing functionality |
| **Private members** | Not inherited |
| **Constructor** | Not inherited, but called |

---

## üìä Inheritance Types

| Type | Description | Example |
|------|-------------|---------|
| **Single** | One parent, one child | Animal ‚Üí Dog |
| **Multilevel** | Chain of inheritance | Vehicle ‚Üí Car ‚Üí ElectricCar |
| **Hierarchical** | One parent, multiple children | Shape ‚Üí Circle, Rectangle |
| **Multiple** | Multiple parents (NOT supported) | Use interfaces |
| **Hybrid** | Mix of types (NOT supported) | Use interfaces |

## üí° Best Practices

1. ‚úÖ **Use inheritance for IS-A relationship**
   ```java
   class Dog extends Animal { }  // Dog IS-A Animal
   ```

2. ‚úÖ **Override methods with @Override annotation**
   ```java
   @Override
   void display() { }  // Catch errors at compile-time
   ```

3. ‚úÖ **Call super() in child constructor**
   ```java
   Child(int x) {
       super(x);  // Initialize parent
   }
   ```

4. ‚úÖ **Don't break parent class contract**
   ```java
   // Maintain expected behavior
   ```

5. ‚úÖ **Prefer composition over inheritance when appropriate**
   ```java
   // Has-A is often better than IS-A
   ```

## ‚ö†Ô∏è Common Pitfalls

```java
public class InheritancePitfalls {
    // 1. Private members not inherited
    class Parent {
        private int x = 10;
    }
    class Child extends Parent {
        void display() {
            // System.out.println(x);  // ERROR!
        }
    }
    
    // 2. Constructor not inherited
    class Parent2 {
        Parent2(int x) { }
    }
    class Child2 extends Parent2 {
        Child2() {
            super(0);  // Must call parent constructor
        }
    }
    
    // 3. Method hiding (static methods)
    class Parent3 {
        static void display() {
            System.out.println("Parent");
        }
    }
    class Child3 extends Parent3 {
        static void display() {  // Hiding, not overriding
            System.out.println("Child");
        }
    }
    
    // 4. Final methods cannot be overridden
    class Parent4 {
        final void display() { }
    }
    class Child4 extends Parent4 {
        // void display() { }  // ERROR!
    }
}
```
