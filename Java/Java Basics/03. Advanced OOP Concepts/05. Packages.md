# üì¶ Packages

## üìñ Overview

A **package** in Java is a namespace that organizes related classes and interfaces. It helps avoid naming conflicts and makes code more maintainable and modular.

**Benefits:**
- ‚úÖ Namespace management (avoid name conflicts)
- ‚úÖ Access control
- ‚úÖ Code organization
- ‚úÖ Easier to locate related classes
- ‚úÖ Reusability

---

## üéØ Package Types

```
1. Built-in Packages (java.*, javax.*)
   - java.lang (default, auto-imported)
   - java.util
   - java.io
   - java.net
   - java.sql
   
2. User-defined Packages
   - Created by developers
```

## üíª Example 1: Creating and Using Package

**File: com/company/utils/Calculator.java**
```java
package com.company.utils;

public class Calculator {
    public int add(int a, int b) {
        return a + b;
    }
    
    public int subtract(int a, int b) {
        return a - b;
    }
    
    public int multiply(int a, int b) {
        return a * b;
    }
    
    public double divide(int a, int b) {
        if (b == 0) {
            throw new ArithmeticException("Cannot divide by zero");
        }
        return (double) a / b;
    }
}
```

**File: com/company/app/Main.java**
```java
package com.company.app;

import com.company.utils.Calculator;

public class Main {
    public static void main(String[] args) {
        Calculator calc = new Calculator();
        
        System.out.println("10 + 5 = " + calc.add(10, 5));
        System.out.println("10 - 5 = " + calc.subtract(10, 5));
        System.out.println("10 * 5 = " + calc.multiply(10, 5));
        System.out.println("10 / 5 = " + calc.divide(10, 5));
    }
}
```

## üíª Example 3: Import Statements

```java
package com.example.demo;

// 1. Import specific class
import java.util.ArrayList;

// 2. Import all classes from package (not recommended for production)
import java.util.*;

// 3. Import static members
import static java.lang.Math.PI;
import static java.lang.Math.sqrt;

// 4. Import specific static method
import static java.lang.System.out;

public class ImportDemo {
    public static void main(String[] args) {
        // Using imported classes
        ArrayList<String> list = new ArrayList<>();
        list.add("Hello");
        list.add("World");
        
        HashMap<String, Integer> map = new HashMap<>();
        map.put("one", 1);
        map.put("two", 2);
        
        // Using static imports
        out.println("List: " + list);
        out.println("Map: " + map);
        out.println("PI = " + PI);
        out.println("Square root of 16 = " + sqrt(16));
    }
}
```

## üíª Example 4: Fully Qualified Names

```java
package com.example.app;

public class FullyQualifiedDemo {
    public static void main(String[] args) {
        // Without import - using fully qualified names
        java.util.ArrayList<String> list = new java.util.ArrayList<>();
        list.add("Item 1");
        list.add("Item 2");
        
        java.util.Date date = new java.util.Date();
        
        System.out.println("List: " + list);
        System.out.println("Date: " + date);
        
        // Useful when classes have same name from different packages
        java.sql.Date sqlDate = new java.sql.Date(System.currentTimeMillis());
        java.util.Date utilDate = new java.util.Date();
        
        System.out.println("SQL Date: " + sqlDate);
        System.out.println("Util Date: " + utilDate);
    }
}
```

## üíª Example 7: Access Control with Packages

**File: com/company/data/DatabaseConfig.java**
```java
package com.company.data;

public class DatabaseConfig {
    // Public - accessible from anywhere
    public String url = "jdbc:mysql://localhost:3306/mydb";
    
    // Protected - accessible within package and subclasses
    protected String username = "admin";
    
    // Default (package-private) - accessible within package only
    String password = "secret";
    
    // Private - accessible within class only
    private String encryptionKey = "key123";
    
    public void displayPublic() {
        System.out.println("Public: URL = " + url);
    }
    
    protected void displayProtected() {
        System.out.println("Protected: Username = " + username);
    }
    
    void displayDefault() {
        System.out.println("Default: Password = " + password);
    }
    
    private void displayPrivate() {
        System.out.println("Private: Key = " + encryptionKey);
    }
}
```

**File: com/company/data/DataManager.java (same package)**
```java
package com.company.data;

public class DataManager {
    public void accessConfig() {
        DatabaseConfig config = new DatabaseConfig();
        
        // Can access public, protected, and default
        System.out.println(config.url);         // OK - public
        System.out.println(config.username);    // OK - protected
        System.out.println(config.password);    // OK - default (same package)
        // System.out.println(config.encryptionKey);  // ERROR - private
        
        config.displayPublic();
        config.displayProtected();
        config.displayDefault();
        // config.displayPrivate();  // ERROR - private
    }
}
```

**File: com/company/app/Application.java (different package)**
```java
package com.company.app;

import com.company.data.DatabaseConfig;

public class Application {
    public static void main(String[] args) {
        DatabaseConfig config = new DatabaseConfig();
        
        // Can only access public members
        System.out.println(config.url);  // OK - public
        // System.out.println(config.username);  // ERROR - protected (not subclass)
        // System.out.println(config.password);  // ERROR - default
        // System.out.println(config.encryptionKey);  // ERROR - private
        
        config.displayPublic();  // OK
        // config.displayProtected();  // ERROR
        // config.displayDefault();  // ERROR
    }
}
```

## üíª Example 8: Built-in Packages

```java
package com.example.builtin;

// java.lang - automatically imported
// No need to import String, System, Math, etc.

// java.util - collections, date, random
import java.util.ArrayList;
import java.util.HashMap;
import java.util.Date;
import java.util.Random;

// java.io - input/output
import java.io.File;
import java.io.FileWriter;
import java.io.IOException;

// java.net - networking
import java.net.URL;
import java.net.URLConnection;

public class BuiltInPackagesDemo {
    public static void main(String[] args) {
        // java.lang (auto-imported)
        String str = "Hello";
        System.out.println(str);
        System.out.println("Square root: " + Math.sqrt(16));
        
        // java.util
        ArrayList<Integer> list = new ArrayList<>();
        list.add(1);
        list.add(2);
        System.out.println("List: " + list);
        
        Date date = new Date();
        System.out.println("Date: " + date);
        
        Random random = new Random();
        System.out.println("Random: " + random.nextInt(100));
        
        // java.io
        File file = new File("test.txt");
        System.out.println("File exists: " + file.exists());
    }
}
```

## üíª Example 9: Package Naming Conventions

```java
// Good package naming conventions

// 1. All lowercase
package com.company.project;

// 2. Reverse domain name
package com.google.search;
package org.apache.commons;

// 3. Hierarchical structure
package com.company.product.model;
package com.company.product.service;
package com.company.product.controller;

// 4. Meaningful names
package com.school.student.management;
package com.hospital.patient.records;

// Bad examples:
// package Com.Company.Project;  // Should be lowercase
// package myPackage;            // Not descriptive enough
// package test123;              // Poor naming
```

**File: com/ecommerce/product/service/ProductService.java**

```java
package com.ecommerce.product.service;

import com.ecommerce.product.model.Product;
import java.util.ArrayList;
import java.util.List;

public class ProductService {
    private List<Product> products = new ArrayList<>();
    
    public void addProduct(Product product) {
        products.add(product);
        System.out.println("Product added: " + product.getName());
    }
    
    public void displayAll() {
        System.out.println("=== All Products ===");
        for (Product p : products) {
            System.out.println(p.getId() + ": " + p.getName() + 
                             " - $" + p.getPrice());
        }
    }
}
```

**File: com/ecommerce/Main.java**
```java
package com.ecommerce;

import com.ecommerce.product.model.Product;

    public static void main(String[] args) {
        ProductService service = new ProductService();

        service.addProduct(new Product(2, "Mouse", 29.99));
        service.addProduct(new Product(3, "Keyboard", 79.99));
```

## üìä Import Types

| Import Type | Syntax | Example | When to Use |
|-------------|--------|---------|-------------|
| **Single class** | `import package.Class;` | `import java.util.ArrayList;` | Preferred |
| **All classes** | `import package.*;` | `import java.util.*;` | Use sparingly |
| **Static member** | `import static package.Class.member;` | `import static Math.PI;` | For constants |
| **Static all** | `import static package.Class.*;` | `import static Math.*;` | Rarely |

## üí° Best Practices

1. ‚úÖ **Use reverse domain name**
   ```java
   package com.company.project;
   ```

2. ‚úÖ **Use lowercase for package names**
   ```java
   package com.example.util;
   ```

3. ‚úÖ **Organize by feature, not layer**
   ```java
   // Good
   com.app.user.model, com.app.user.service
   
   // Less good
   com.app.model.user, com.app.service.user
   ```

4. ‚úÖ **Import specific classes**
   ```java
   import java.util.ArrayList;  // Good
   // import java.util.*;        // Less preferred
   ```

5. ‚úÖ **One public class per file**
   ```java
   // File: Student.java
   package com.school;
   public class Student { }
   ```

## ‚ö†Ô∏è Common Pitfalls

```java
// 1. Package statement must be first (except comments)
// import java.util.*;  // ERROR!
package com.example;    // Must be first
import java.util.*;

// 2. Package name must match directory structure
// File location: com/example/Test.java
package com.example;  // Correct
// package com.wrong;   // ERROR!

// 3. Cannot import default package
// If A.java is in default package
// package com.example;
// import A;  // ERROR! Cannot import from default package

// 4. Classes in different packages must import
package com.example.app;
// import com.example.util.Helper;  // Must import
// Helper helper = new Helper();    // ERROR without import

// 5. java.lang is auto-imported
package com.example;
// No need to import String, System, Math, etc.
```
