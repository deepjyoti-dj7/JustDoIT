### Q1: What is Java? What are its main features?

**Answer:** Java is a high-level, object-oriented programming language.

**Key Features:**
- Platform independent (Write Once, Run Anywhere)
- Object-oriented
- Simple and familiar syntax
- Automatic memory management (Garbage Collection)
- Multithreading support

### Q2: Explain JDK, JRE, and JVM

**Answer:**
- **JVM (Java Virtual Machine):** Executes Java bytecode, provides platform independence
- **JRE (Java Runtime Environment):** JVM + Libraries needed to run Java applications
- **JDK (Java Development Kit):** JRE + Development tools (compiler, debugger)

```
JDK = JRE + Development Tools
JRE = JVM + Library Classes
```

### Q3: What is the difference between `==` and `equals()`?

**Answer:**
- `==` compares **references** (memory addresses)
- `equals()` compares **values** (content)

```java
String s1 = new String("Hello");
String s2 = new String("Hello");

s1 == s2       // false (different objects)
s1.equals(s2)  // true (same content)

Integer a = 127;
Integer b = 127;
a == b         // true (Integer cache -128 to 127)

Integer x = 128;
Integer y = 128;
x == y         // false (outside cache range)
```

### Q4: What is autoboxing and unboxing?

**Answer:**
- **Autoboxing:** Automatic conversion of primitive to wrapper class
- **Unboxing:** Automatic conversion of wrapper class to primitive

```java
// Autoboxing
Integer obj = 10;  // int -> Integer

// Unboxing
int value = obj;   // Integer -> int

// In collections
List<Integer> list = new ArrayList<>();
list.add(5);       // Autoboxing: int -> Integer
int x = list.get(0);  // Unboxing: Integer -> int
```

### Q6: What is the difference between String, StringBuilder, and StringBuffer?

**Answer:**

| Feature | String | StringBuilder | StringBuffer |
|---------|--------|---------------|--------------|
| **Mutability** | Immutable | Mutable | Mutable |
| **Thread-safe** | Yes | No | Yes |
| **Performance** | Slow | Fast | Moderate |
| **Use case** | Fixed strings | Single-thread | Multi-thread |

```java
// String - creates new object
String s = "Hello";
s = s + " World";  // New object created

// StringBuilder - modifies same object (fast)
StringBuilder sb = new StringBuilder("Hello");
sb.append(" World");  // Same object modified

// StringBuffer - thread-safe (slower than StringBuilder)
StringBuffer sbf = new StringBuffer("Hello");
sbf.append(" World");
```

### Q7: What is method overloading and overriding?

**Answer:**

**Overloading (Compile-time polymorphism):**
- Same method name, different parameters
- In same class

```java
class Calculator {
    int add(int a, int b) { return a + b; }
    double add(double a, double b) { return a + b; }
    int add(int a, int b, int c) { return a + b + c; }
}
```

**Overriding (Runtime polymorphism):**
- Same method signature in subclass
- Changes parent class behavior

```java
class Animal {
    void sound() { System.out.println("Animal sound"); }
}

class Dog extends Animal {
    @Override
    void sound() { System.out.println("Bark"); }
}
```

### Q8: What is the difference between abstract class and interface?

**Answer:**

| Feature | Abstract Class | Interface |
|---------|----------------|-----------|
| **Methods** | Abstract + Concrete | Abstract + Default (Java 8+) |
| **Variables** | Any type | public static final only |
| **Constructor** | Yes | No |
| **Inheritance** | Single | Multiple |
| **Access modifiers** | Any | public only (methods) |
| **When to use** | IS-A relationship | CAN-DO capability |

```java
// Abstract class
abstract class Animal {
    String name;
    abstract void sound();
    void sleep() { System.out.println("Sleeping"); }
}

// Interface
interface Flyable {
    void fly();
    default void glide() { System.out.println("Gliding"); }
}
```

### Q9: What is static keyword?

**Answer:** `static` belongs to class, not instance.

```java
class Counter {
    static int count = 0;        // Shared by all instances
    int instanceCount = 0;       // Unique to each instance
    
    static void increment() {    // Can be called without object
        count++;
    }
    
    static {                     // Static block - runs once
        System.out.println("Class loaded");
    }
}

// Usage
Counter.increment();  // Call without object
System.out.println(Counter.count);
```

**Key Points:**
- Static variables shared by all instances
- Static methods can't access instance variables
- Static blocks execute when class loads

### Q10: Explain garbage collection

**Answer:** Automatic memory management that reclaims unused objects.

**How it works:**
1. Mark live objects (reachable from GC roots)
2. Sweep dead objects
3. Compact memory (optional)

**When object is eligible for GC:**
```java
// 1. Null reference
Object obj = new Object();
obj = null;  // Eligible

// 2. Reassigned reference
String s = "Hello";
s = "World";  // "Hello" eligible

// 3. Object created in method
void method() {
    Object local = new Object();
}  // local eligible after method returns
```

**GC Methods:**
```java
System.gc();           // Request GC (not guaranteed)
Runtime.getRuntime().gc();  // Alternative
```

### Q11: What is constructor chaining?

**Answer:** Calling one constructor from another in same class or parent class.

```java
class Employee {
    private String name;
    private int age;
    private String department;
    
    // Constructor 1
    public Employee() {
        this("Unknown", 0);  // Calls constructor 2
    }
    
    // Constructor 2
    public Employee(String name, int age) {
        this(name, age, "General");  // Calls constructor 3
    }
    
    // Constructor 3
    public Employee(String name, int age, String department) {
        this.name = name;
        this.age = age;
        this.department = department;
    }
}

// Parent-child chaining
class Parent {
    Parent() {
        System.out.println("Parent constructor");
    }
}

class Child extends Parent {
    Child() {
        super();  // Calls parent constructor
        System.out.println("Child constructor");
    }
}
```

### Q13: What is the difference between `this` and `super`?

**Answer:**

| Keyword | Usage |
|---------|-------|
| **this** | Refers to current class instance |
| **super** | Refers to parent class instance |

```java
class Parent {
    String name = "Parent";
    void display() { System.out.println("Parent"); }
}

class Child extends Parent {
    String name = "Child";
    
    void show() {
        System.out.println(this.name);   // Child
        System.out.println(super.name);  // Parent
        
        this.display();   // Calls Child's display (if overridden)
        super.display();  // Calls Parent's display
    }
}
```

### Q15: What is the difference between `Comparable` and `Comparator`?

**Answer:**

| Feature | Comparable | Comparator |
|---------|-----------|------------|
| **Package** | java.lang | java.util |
| **Method** | compareTo(T obj) | compare(T o1, T o2) |
| **Usage** | Natural ordering | Custom ordering |
| **Modification** | Modify class | External class |

```java
// Comparable - natural ordering
class Student implements Comparable<Student> {
    String name;
    int marks;
    
    @Override
    public int compareTo(Student other) {
        return this.marks - other.marks;  // Sort by marks
    }
}

// Comparator - custom ordering
class NameComparator implements Comparator<Student> {
    @Override
    public int compare(Student s1, Student s2) {
        return s1.name.compareTo(s2.name);  // Sort by name
    }
}

// Usage
Collections.sort(students);  // Uses Comparable
Collections.sort(students, new NameComparator());  // Uses Comparator
```
