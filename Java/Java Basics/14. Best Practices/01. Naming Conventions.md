# üìù Naming Conventions

## üìñ Overview

**Naming conventions** ensure code readability and maintainability. Java follows specific naming standards for different code elements.

---

## üíª Class Names

```java
// ‚úÖ PascalCase (UpperCamelCase)
public class Customer { }
public class ShoppingCart { }
public class OrderProcessor { }
public class UserAccountManager { }

// ‚ùå Incorrect
public class customer { }           // lowercase
public class shopping_cart { }      // snake_case
public class orderprocessor { }     // no separation
```

**Rules:**
- Use **PascalCase** (first letter uppercase)
- Nouns or noun phrases
- Descriptive and meaningful
- Avoid abbreviations unless widely known

---

## üíª Interface Names

```java
// ‚úÖ PascalCase, often adjectives or nouns
public interface Serializable { }
public interface Runnable { }
public interface Comparable<T> { }
public interface PaymentProcessor { }
public interface DataSource { }

// Common patterns
public interface CanFly { }         // Ability
public interface Clickable { }      // Adjective
public interface Repository { }     // Noun

// ‚ùå Avoid
public interface IPayment { }       // Hungarian notation (avoid 'I' prefix)
```

## üíª Method Names

```java
public class Example {
    // ‚úÖ camelCase, start with lowercase
    public void calculateTotal() { }
    public String getUserName() { }
    public boolean isActive() { }
    public void setEmail(String email) { }
    
    // Boolean methods - use is/has/can/should
    public boolean isEmpty() { }
    public boolean hasPermission() { }
    public boolean canEdit() { }
    public boolean shouldRetry() { }
    
    // ‚ùå Incorrect
    public void CalculateTotal() { }     // PascalCase
    public void calculate_total() { }    // snake_case
    public void calcTot() { }            // unclear abbreviation
}
```

**Rules:**
- Use **camelCase** (first letter lowercase)
- Verbs or verb phrases
- Descriptive action names
- Getters: `getXxx()`, Booleans: `isXxx()`, Setters: `setXxx()`

## üíª Variable Names

```java
public class VariableNaming {
    // ‚úÖ camelCase for variables
    private String firstName;
    private int accountNumber;
    private boolean isActive;
    private List<String> emailAddresses;
    
    // Local variables
    public void processOrder() {
        int totalAmount = 0;
        String customerName = "Alice";
        boolean isValid = true;
    }
    
    // Loop variables
    for (int i = 0; i < 10; i++) { }           // OK for simple loops
    for (String item : items) { }              // Descriptive
    for (Customer customer : customers) { }    // Descriptive
    
    // ‚ùå Incorrect
    private String FirstName;        // PascalCase
    private int account_number;      // snake_case
    private boolean b;               // Single letter (except loops)
    private List<String> ea;         // Unclear abbreviation
}
```

## üíª Constants

```java

    // ‚úÖ UPPER_SNAKE_CASE

    public static final String DEFAULT_LANGUAGE = "English";
    public static final double PI = 3.14159;
    public static final String DATABASE_URL = "jdbc:mysql://localhost:3306";

        ACTIVE,

        PENDING
    }
    
    // ‚ùå Incorrect

    public static final String default_language;     // mixed case
```

## üíª Package Names

```java
// ‚úÖ lowercase, separated by dots
package com.company.project;
package com.example.ecommerce.service;
package org.springframework.boot;
package java.util.concurrent;

// ‚ùå Incorrect
package com.Company.Project;         // PascalCase
package com.example.eCommerce;       // mixed case
package com.example.e_commerce;      // underscore
```

**Rules:**
- All **lowercase**
- Reverse domain name convention
- Use dots to separate levels
- No underscores or special characters

## üìä Naming Convention Summary

| Element | Convention | Example |
|---------|-----------|---------|
| **Class** | PascalCase | `Customer`, `OrderProcessor` |
| **Interface** | PascalCase | `Runnable`, `Comparable` |
| **Method** | camelCase | `calculateTotal()`, `isActive()` |
| **Variable** | camelCase | `firstName`, `accountNumber` |
| **Constant** | UPPER_SNAKE_CASE | `MAX_SIZE`, `DEFAULT_VALUE` |
| **Package** | lowercase | `com.example.project` |
| **Enum Type** | PascalCase | `OrderStatus` |
| **Enum Constant** | UPPER_SNAKE_CASE | `PENDING`, `ACTIVE` |
| **Generic** | Single letter | `T`, `E`, `K`, `V` |

---

## üìä Boolean Naming

| Prefix | Usage | Example |
|--------|-------|---------|
| **is** | State/condition | `isActive`, `isEmpty`, `isValid` |
| **has** | Possession | `hasPermission`, `hasChildren` |
| **can** | Ability | `canEdit`, `canDelete`, `canAccess` |
| **should** | Recommendation | `shouldRetry`, `shouldCache` |

---

## ‚ö° Best Practices

| Practice | Description |
|----------|-------------|
| ‚úÖ Be descriptive | `getUserById()` not `get()` |
| ‚úÖ Be consistent | Follow same patterns throughout |
| ‚úÖ Avoid abbreviations | `customer` not `cust` |
| ‚úÖ Use domain language | Match business terminology |
| ‚úÖ Avoid noise words | No `TheCustomer`, `CustomerInfo` |
| ‚úÖ Searchable names | Easy to find in codebase |
| ‚ùå No single letters | Except `i`, `j`, `k` in loops |
| ‚ùå No Hungarian notation | No `strName`, `intAge` |
| ‚ùå No magic numbers | Use named constants |

## üíª Real-World Examples

```java
// ‚úÖ Good naming
public class OrderService {
    private static final int MAX_RETRY_ATTEMPTS = 3;
    private static final String DEFAULT_CURRENCY = "USD";
    
    private OrderRepository orderRepository;
    private PaymentProcessor paymentProcessor;
    
    public Order createOrder(Customer customer, List<OrderItem> items) {
        if (!isValidCustomer(customer)) {
            throw new InvalidCustomerException();
        }
        
        double totalAmount = calculateTotalAmount(items);
        Order order = new Order(customer, items, totalAmount);
        
        return orderRepository.save(order);
    }
    
    private boolean isValidCustomer(Customer customer) {
        return customer != null && customer.hasActiveAccount();
    }
    
    private double calculateTotalAmount(List<OrderItem> items) {
        return items.stream()
            .mapToDouble(OrderItem::getPrice)
            .sum();
    }
}

// ‚ùå Poor naming
public class OS {
    private static final int MAX = 3;
    private static final String DEF = "USD";

    private PP pp;
    
    public O cO(C c, List<OI> is) {

        }

        O o = new O(c, is, ta);
        
        return or.s(o);
    }
    
    private boolean vC(C c) {
        return c != null && c.hAA();
    }
    
    private double cTA(List<OI> is) {
        return is.stream()
            .mapToDouble(OI::gP)

    }
}
```

## üéØ Context-Specific Naming

```java
// Collection names - plural
List<Customer> customers;
Set<String> emailAddresses;
Map<String, Order> ordersByOrderId;

// Stream operations - descriptive
List<String> activeCustomerNames = customers.stream()
    .filter(customer -> customer.isActive())
    .map(customer -> customer.getName())
    .collect(Collectors.toList());

// Temporary variables - still meaningful
for (Customer tempCustomer : customers) {
    processCustomer(tempCustomer);
}

// Exception classes - end with Exception
public class InvalidOrderException extends Exception { }
public class PaymentFailedException extends RuntimeException { }
```

## üí° Key Takeaways

- Classes/Interfaces: PascalCase
- Methods/Variables: camelCase
- Constants: UPPER_SNAKE_CASE
- Packages: lowercase
- Use descriptive, meaningful names
- Boolean methods: is/has/can/should prefix
- Avoid abbreviations and single letters
- Be consistent across codebase
- Names should reveal intent
- Follow Java standard conventions
