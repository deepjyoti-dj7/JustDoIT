# Cache Configuration - Complete Guide

---

## 1. Introduction to Cache Configuration

Spring Boot provides flexible cache configuration through properties, Java configuration, and external files. Proper configuration ensures optimal performance and resource utilization.

### Configuration Options
- **Application Properties** - YAML/properties files
- **Java Configuration** - @Configuration classes
- **External Config Files** - ehcache.xml, etc.
- **Programmatic Configuration** - Runtime setup

---

## 2. Spring Boot Auto-Configuration

### 2.1. Detection Order

Spring Boot auto-configures cache providers in this order:
1. **Generic** (custom CacheManager bean)
2. **JCache (JSR-107)** (EhCache, Hazelcast)
3. **EhCache 2.x**
4. **Hazelcast**
5. **Infinispan**
6. **Couchbase**
7. **Redis**
8. **Caffeine**
9. **Simple** (ConcurrentHashMap)

### 2.2. Forcing a Provider

```yaml
spring:
  cache:
    type: caffeine  # Forces Caffeine even if Redis is available
```

## 4. Caffeine Configuration

### 4.1. Properties-Based

```yaml
spring:
  cache:
    type: caffeine
    caffeine:
      spec: maximumSize=1000,expireAfterWrite=10m,recordStats
    cache-names:
      - products
      - users
```

### 4.2. Java Configuration

```java
@Configuration
@EnableCaching
public class CaffeineCacheConfig {
    
    @Value("${cache.caffeine.max-size:1000}")
    private int maxSize;
    
    @Value("${cache.caffeine.ttl-minutes:10}")
    private int ttlMinutes;
    
    @Bean
    public CacheManager cacheManager() {
        CaffeineCacheManager cacheManager = new CaffeineCacheManager();
        cacheManager.setCaffeine(caffeineCacheBuilder());
        cacheManager.setCacheNames(Arrays.asList("products", "users", "orders"));
        return cacheManager;
    }
    
    @Bean
    public Caffeine<Object, Object> caffeineCacheBuilder() {
        return Caffeine.newBuilder()
                .initialCapacity(100)
                .maximumSize(maxSize)
                .expireAfterWrite(ttlMinutes, TimeUnit.MINUTES)
                .recordStats();
    }
}
```

### 4.3. Per-Cache Configuration

```java
@Configuration
public class CustomCaffeineCacheConfig {
    
    @Bean
    public CacheManager cacheManager() {
        SimpleCacheManager cacheManager = new SimpleCacheManager();
        
        List<CaffeineCache> caches = Arrays.asList(
            buildCache("products", 5000, 30),
            buildCache("users", 1000, 10),
            buildCache("sessions", 500, 60),
            buildCache("shortLived", 100, 1)
        );
        
        cacheManager.setCaches(caches);
        return cacheManager;
    }
    
    private CaffeineCache buildCache(String name, int maxSize, int ttlMinutes) {
        return new CaffeineCache(name, Caffeine.newBuilder()
            .maximumSize(maxSize)
            .expireAfterWrite(ttlMinutes, TimeUnit.MINUTES)
            .recordStats()
            .build());
    }
}
```

## 5. Redis Configuration

### 5.1. Properties-Based

```yaml
spring:
  cache:
    type: redis
    redis:
      time-to-live: 600000      # 10 minutes (ms)
      cache-null-values: false
      key-prefix: "myapp::"
      use-key-prefix: true
  
  data:
    redis:
      host: localhost
      port: 6379
      password: ${REDIS_PASSWORD:}
      database: 0
      timeout: 2000ms
      connect-timeout: 2000ms
      
      lettuce:
        pool:
          max-active: 8
          max-idle: 8
          min-idle: 2
          max-wait: -1ms
        shutdown-timeout: 100ms
```

### 5.2. Java Configuration

```java
@Configuration
@EnableCaching
public class RedisCacheConfig {
    
    @Value("${cache.redis.ttl-minutes:10}")
    private int ttlMinutes;
    
    @Bean
    public RedisCacheManager cacheManager(RedisConnectionFactory connectionFactory) {
        RedisCacheConfiguration defaultConfig = RedisCacheConfiguration.defaultCacheConfig()
                .entryTtl(Duration.ofMinutes(ttlMinutes))
                .serializeKeysWith(RedisSerializationContext.SerializationPair
                    .fromSerializer(new StringRedisSerializer()))
                .serializeValuesWith(RedisSerializationContext.SerializationPair
                    .fromSerializer(new GenericJackson2JsonRedisSerializer()))
                .disableCachingNullValues()
                .prefixCacheNameWith("myapp::");
        
        return RedisCacheManager.builder(connectionFactory)
                .cacheDefaults(defaultConfig)
                .transactionAware()
                .build();
    }
}
```

### 5.3. Per-Cache TTL

```java
@Bean
public RedisCacheManager cacheManager(RedisConnectionFactory connectionFactory) {
    Map<String, RedisCacheConfiguration> cacheConfigurations = new HashMap<>();
    
    // Products: 1 hour
    cacheConfigurations.put("products", 
        RedisCacheConfiguration.defaultCacheConfig()
            .entryTtl(Duration.ofHours(1))
            .prefixCacheNameWith("product::"));
    
    // Users: 30 minutes
    cacheConfigurations.put("users",
        RedisCacheConfiguration.defaultCacheConfig()
            .entryTtl(Duration.ofMinutes(30))
            .prefixCacheNameWith("user::"));
    
    // Sessions: 24 hours
    cacheConfigurations.put("sessions",
        RedisCacheConfiguration.defaultCacheConfig()
            .entryTtl(Duration.ofHours(24))
            .prefixCacheNameWith("session::"));
    
    // Short-lived: 1 minute
    cacheConfigurations.put("shortLived",
        RedisCacheConfiguration.defaultCacheConfig()
            .entryTtl(Duration.ofMinutes(1)));
    
    return RedisCacheManager.builder(connectionFactory)
            .cacheDefaults(RedisCacheConfiguration.defaultCacheConfig()
                .entryTtl(Duration.ofMinutes(10)))
            .withInitialCacheConfigurations(cacheConfigurations)
            .transactionAware()
            .build();
}
```

## 6. Time-to-Live (TTL) Configuration

### 6.1. Global TTL

```java
@Bean
public CacheManager cacheManager(RedisConnectionFactory factory) {
    return RedisCacheManager.builder(factory)
        .cacheDefaults(RedisCacheConfiguration.defaultCacheConfig()
            .entryTtl(Duration.ofMinutes(10)))
        .build();
}
```

### 6.2. Per-Entry TTL (Advanced)

```java
@Component
public class CustomRedisCacheWriter {
    
    @Autowired
    private RedisTemplate<String, Object> redisTemplate;
    
    public void cacheWithTtl(String cacheName, String key, Object value, Duration ttl) {
        String fullKey = cacheName + "::" + key;
        redisTemplate.opsForValue().set(fullKey, value, ttl);
    }
}

// Usage
@Service
public class ProductService {
    
    @Autowired
    private CustomRedisCacheWriter cacheWriter;
    
    public Product getProduct(Long id) {
        // Check cache
        String key = "product::" + id;
        Product product = (Product) redisTemplate.opsForValue().get(key);
        
        if (product == null) {
            // Load from DB
            product = productRepository.findById(id).orElseThrow();
            
            // Cache with custom TTL based on product type
            Duration ttl = product.isPopular() 
                ? Duration.ofHours(2) 
                : Duration.ofMinutes(10);
            
            cacheWriter.cacheWithTtl("products", id.toString(), product, ttl);
        }
        
        return product;
    }
}
```

```java
@Bean
public Caffeine<Object, Object> caffeineCacheBuilder() {
    return Caffeine.newBuilder()
        .maximumWeight(10000)
        .weigher((key, value) -> {

                return ((String) value).length();

            return 1;
```

## 8. Eviction Policies

### 8.1. Caffeine Eviction

```java
@Bean
public Caffeine<Object, Object> caffeineCacheBuilder() {
    return Caffeine.newBuilder()
        .maximumSize(1000)
        .expireAfterWrite(10, TimeUnit.MINUTES)  // Write-based
        .expireAfterAccess(5, TimeUnit.MINUTES)  // Access-based
        .build();
}
```

### 8.2. Redis Eviction

Configure in `redis.conf`:
- **noeviction** - Return errors when memory limit reached
- **allkeys-lru** - Remove any key using LRU
- **allkeys-lfu** - Remove any key using LFU
- **volatile-lru** - Remove keys with expiration using LRU
- **volatile-lfu** - Remove keys with expiration using LFU
- **allkeys-random** - Remove random keys
- **volatile-random** - Remove random keys with expiration
- **volatile-ttl** - Remove keys with shortest TTL

```properties
# redis.conf
maxmemory-policy allkeys-lru
```

## 10. Multiple Cache Managers

### 10.1. Configuration

```java
@Configuration
@EnableCaching
public class MultiCacheConfig {
    
    @Bean
    @Primary
    public CacheManager primaryCacheManager() {
        CaffeineCacheManager cacheManager = new CaffeineCacheManager();
        cacheManager.setCaffeine(Caffeine.newBuilder()
            .maximumSize(1000)
            .expireAfterWrite(10, TimeUnit.MINUTES));
        return cacheManager;
    }
    
    @Bean("redisCacheManager")
    public CacheManager redisCacheManager(RedisConnectionFactory factory) {
        return RedisCacheManager.builder(factory)
            .cacheDefaults(RedisCacheConfiguration.defaultCacheConfig()
                .entryTtl(Duration.ofHours(1)))
            .build();
    }
}
```

### 10.2. Usage

```java
@Service
public class ProductService {
    
    // Uses primary (Caffeine)
    @Cacheable("products")
    public Product getProduct(Long id) {
        return productRepository.findById(id).orElseThrow();
    }
    
    // Uses Redis
    @Cacheable(value = "popularProducts", cacheManager = "redisCacheManager")
    public List<Product> getPopularProducts() {
        return productRepository.findPopular();
    }
}
```

## 11. Cache Key Configuration

### 11.1. Custom Key Generator

```java
@Configuration
public class CacheKeyConfig {
    
    @Bean("customKeyGenerator")
    public KeyGenerator keyGenerator() {
        return (target, method, params) -> {
            StringBuilder key = new StringBuilder();
            key.append(target.getClass().getSimpleName());
            key.append("_").append(method.getName());
            for (Object param : params) {
                key.append("_").append(param.toString());
            }
            return key.toString();
        };
    }
}
```

### 11.2. Usage

```java
@Cacheable(value = "products", keyGenerator = "customKeyGenerator")
public Product getProduct(Long id, String locale) {
    return productRepository.findByIdAndLocale(id, locale);
}
```

## 14. Transaction-Aware Caching

```java
@Bean
public RedisCacheManager cacheManager(RedisConnectionFactory factory) {
    return RedisCacheManager.builder(factory)
        .cacheDefaults(RedisCacheConfiguration.defaultCacheConfig())
        .transactionAware()  // Enable transaction support
```

### 14.2. Usage

```java
@Service
public class ProductService {
    
    @Transactional
    @CachePut(value = "products", key = "#product.id")

        // Cache update happens after transaction commits
        return productRepository.save(product);
    }
}
```

## 15. Cache Loader Configuration

### 15.1. Caffeine with Cache Loader

```java
public CacheManager cacheManager() {

            return loadFromDatabase(key);
        }
    });
    
    return cacheManager;
}
```

## 16. Best Practices

1. **Use profiles** - Different configs for dev/prod
2. **Set appropriate TTL** - Balance freshness and performance
3. **Configure size limits** - Prevent memory overflow
4. **Enable statistics** - Monitor hit ratio and performance
5. **Use custom key generators** - For complex keys
6. **Configure eviction** - Choose policy based on use case
7. **Transaction-aware** - For consistency with database
8. **Per-cache configuration** - Different TTL/size per cache
9. **Externalize config** - Use properties for flexibility
10. **Monitor and tune** - Adjust based on metrics

## 17. Interview Questions

**Q1: How to configure cache TTL in Spring Boot?**
**A:** Via `spring.cache.redis.time-to-live` or `RedisCacheConfiguration.entryTtl()`.

**Q2: What is the difference between TTL and TTI?**
**A:** TTL: Time since write. TTI: Time since last access.

**Q3: How to configure different TTL per cache?**
**A:** Use `withInitialCacheConfigurations()` in RedisCacheManager.

**Q4: What is cache eviction policy?**
**A:** Algorithm to remove entries when cache is full (LRU, LFU, etc.).

**Q5: How to enable cache statistics?**
**A:** Caffeine: `.recordStats()`. Spring Boot Actuator: `management.metrics.cache.instrument-cache=true`.

**Q6: What is transaction-aware caching?**
**A:** Cache operations participate in database transactions (commit/rollback).

**Q7: How to use multiple cache managers?**
**A:** Define multiple @Bean CacheManager, mark one @Primary, specify in @Cacheable.

**Q8: What is cache key generator?**
**A:** Custom logic to generate cache keys from method parameters.

**Q9: How to configure cache size limit?**
**A:** Caffeine: `maximumSize()`. Redis: `maxmemory` in redis.conf.

**Q10: What is cache warming?**
**A:** Preloading cache with data on application startup.

**Q11: How to configure profile-specific caching?**
**A:** Use `spring.profiles` in application.yml or `@Profile` on beans.

**Q12: What is cache null values?**
**A:** Caching null results to prevent repeated database queries (cache-null-values).

**Q13: How to configure Redis connection pool?**
**A:** Via `spring.data.redis.lettuce.pool.*` properties.

**Q14: What is cache key prefix?**
**A:** Prefix added to all cache keys (e.g., "myapp::products::123").

**Q15: How to disable caching conditionally?**
**A:** Use `@Cacheable(condition = "...")` with SpEL expression.

**Q16: What is cache serialization?**
**A:** Converting objects to bytes for storage. Configure via SerializationPair.

**Q17: How to configure cache for testing?**
**A:** Use `@AutoConfigureCache` or simple ConcurrentMapCacheManager.

**Q18: What is cache loader?**
**A:** Function to load value from source when cache miss occurs.

**Q19: How to configure maximum weight?**
**A:** Caffeine: `maximumWeight()` with custom `weigher()`.

**Q20: What is cache stampede prevention?**
**A:** Use `@Cacheable(sync = true)` to synchronize concurrent loads.

Spring Boot cache configuration is flexible: properties files for simple setups, Java configuration for complex scenarios. Configure TTL, size limits, eviction policies, and statistics monitoring. Use profile-specific configuration for different environments. Enable transaction-awareness and custom key generators as needed.
