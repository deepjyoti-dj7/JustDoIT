## ğŸ”— Security Filter Chain

### Filter Chain Order

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {
    
    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            // 1. CORS
            .cors(cors -> cors.configurationSource(corsConfigurationSource()))
            
            // 2. CSRF
            .csrf(csrf -> csrf.disable())
            
            // 3. Session Management
            .sessionManagement(session -> 
                session.sessionCreationPolicy(SessionCreationPolicy.STATELESS))
            
            // 4. Authorization
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/api/public/**").permitAll()
                .requestMatchers("/api/admin/**").hasRole("ADMIN")
                .anyRequest().authenticated()
            )
            
            // 5. Exception Handling
            .exceptionHandling(ex -> ex
                .authenticationEntryPoint(authenticationEntryPoint()))
            
            // 6. Authentication Filter
            .addFilterBefore(jwtAuthenticationFilter(), 
                UsernamePasswordAuthenticationFilter.class);
        
        return http.build();
    }
}
```

### Common Filters

| Filter | Order | Purpose |
|--------|-------|---------|
| **CorsFilter** | 1 | Handle CORS requests |
| **CsrfFilter** | 2 | CSRF token validation |
| **UsernamePasswordAuthenticationFilter** | 5 | Form login authentication |
| **BasicAuthenticationFilter** | 6 | HTTP Basic authentication |
| **FilterSecurityInterceptor** | Last | Authorization decisions |

### Basic Auth Configuration

```java
@Configuration
@EnableWebSecurity
public class BasicAuthConfig {
    
    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/api/public/**").permitAll()
                .anyRequest().authenticated()
            )
            .httpBasic(Customizer.withDefaults()); // Enable Basic Auth
        
        return http.build();
    }
    
    @Bean
    public InMemoryUserDetailsManager userDetailsService() {
        UserDetails user = User.builder()
            .username("user")
            .password(passwordEncoder().encode("password"))
            .roles("USER")
            .build();
        
        UserDetails admin = User.builder()
            .username("admin")
            .password(passwordEncoder().encode("admin"))
            .roles("ADMIN")
            .build();
        
        return new InMemoryUserDetailsManager(user, admin);
    }
    
    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder();
    }
}
```

## ğŸ“ Form-Based Login

### Configuration

```java
@Configuration
@EnableWebSecurity
public class FormLoginConfig {
    
    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/", "/login", "/css/**", "/js/**").permitAll()
                .anyRequest().authenticated()
            )
            .formLogin(form -> form
                .loginPage("/login")                    // Custom login page
                .loginProcessingUrl("/perform_login")   // Form submit URL
                .defaultSuccessUrl("/home", true)       // Redirect after success
                .failureUrl("/login?error=true")        // Redirect after failure
                .usernameParameter("email")             // Custom username field
                .passwordParameter("pass")              // Custom password field
                .permitAll()
            )
            .logout(logout -> logout
                .logoutUrl("/logout")
                .logoutSuccessUrl("/login?logout=true")
                .deleteCookies("JSESSIONID")
                .invalidateHttpSession(true)
                .permitAll()
            );
        
        return http.build();
    }
}
```

### Custom User Details

```java
// Custom UserDetails implementation
public class CustomUserDetails implements UserDetails {
    
    private final User user;
    
    public CustomUserDetails(User user) {
        this.user = user;
    }
    
    @Override
    public Collection<? extends GrantedAuthority> getAuthorities() {
        return user.getRoles().stream()
            .map(role -> new SimpleGrantedAuthority("ROLE_" + role))
            .collect(Collectors.toList());
    }
    
    @Override
    public String getPassword() {
        return user.getPassword();
    }
    
    @Override
    public String getUsername() {
        return user.getEmail();
    }
    
    @Override
    public boolean isAccountNonExpired() {
        return true;
    }
    
    @Override
    public boolean isAccountNonLocked() {
        return user.isActive();
    }
    
    @Override
    public boolean isCredentialsNonExpired() {
        return true;
    }
    
    @Override
    public boolean isEnabled() {
        return user.isActive();
    }
    
    // Custom getter
    public Long getId() {
        return user.getId();
    }
}
```

### Security Context in Service

```java
@Service
public class AuditService {
    
    public void logAction(String action) {
        Authentication auth = SecurityContextHolder.getContext().getAuthentication();
        
        if (auth != null && auth.isAuthenticated()) {
            String username = auth.getName();
            log.info("User {} performed action: {}", username, action);
        }
    }
    
    public boolean isCurrentUser(Long userId) {
        Authentication auth = SecurityContextHolder.getContext().getAuthentication();
        
        if (auth.getPrincipal() instanceof CustomUserDetails) {
            CustomUserDetails userDetails = (CustomUserDetails) auth.getPrincipal();
            return userDetails.getId().equals(userId);
        }
        
        return false;
    }
}
```


### 4. Use HTTPS in Production


## ğŸ¤ Interview Questions

### Q1: What is Spring Security?
**Answer:** Powerful, customizable authentication and access-control framework for securing Spring applications.

### Q2: Difference between authentication and authorization?
**Answer:**
- **Authentication:** Verifying identity (who are you?)
- **Authorization:** Checking permissions (what can you do?)

### Q3: What is SecurityFilterChain?
**Answer:** Chain of servlet filters that process HTTP requests for security, including authentication, authorization, CSRF protection.

### Q4: What is UserDetailsService?
**Answer:** Core interface to load user-specific data, returns UserDetails containing username, password, authorities.

### Q5: What is PasswordEncoder?
**Answer:** Interface for encoding passwords before storing. Recommended: BCryptPasswordEncoder.

### Q6: How to get current authenticated user?
**Answer:**
```java
Authentication auth = SecurityContextHolder.getContext().getAuthentication();
String username = auth.getName();
```

### Q7: What is @EnableWebSecurity?
**Answer:** Annotation enabling Spring Security's web security support and configuration.

### Q8: What is CSRF?
**Answer:** Cross-Site Request Forgery - attack forcing authenticated users to execute unwanted actions. Spring Security provides CSRF tokens for protection.

### Q9: Difference between hasRole() and hasAuthority()?
**Answer:**
- **hasRole('ADMIN'):** Checks for authority "ROLE_ADMIN" (auto-prefixed)
- **hasAuthority('ADMIN'):** Checks for exact authority "ADMIN"

### Q10: What is SecurityContext?
**Answer:** Thread-local storage holding Authentication object for current request.

### Q11: How to configure in-memory authentication?
**Answer:**
```java
@Bean
public InMemoryUserDetailsManager users() {
    UserDetails user = User.builder()
        .username("user")
        .password(passwordEncoder().encode("password"))
        .roles("USER")
        .build();
    return new InMemoryUserDetailsManager(user);
}
```

### Q12: What is BCrypt?
**Answer:** Strong password hashing algorithm using salts to prevent rainbow table attacks. Recommended for password encoding.

### Q13: How to disable security for specific endpoints?
**Answer:**
```java
.authorizeHttpRequests(auth -> auth
    .requestMatchers("/api/public/**").permitAll()
    .anyRequest().authenticated()
)
```

### Q14: What is Principal?
**Answer:** Interface representing authenticated user, contains user identity information.

### Q15: How to implement custom login page?
**Answer:**
```java
.formLogin(form -> form
    .loginPage("/login")
    .permitAll()
)
```

### Q16: What is session fixation attack?
**Answer:** Attacker sets victim's session ID. Spring Security creates new session after authentication to prevent this.

### Q17: How to force HTTPS?
**Answer:**
```java
http.requiresChannel(channel -> 
    channel.anyRequest().requiresSecure());
```

### Q18: What is @AuthenticationPrincipal?
**Answer:** Annotation to inject current authenticated user's UserDetails into controller method.

### Q19: Default security configuration?
**Answer:** All endpoints secured, form login enabled, HTTP Basic enabled, CSRF protection enabled.

### Q20: Best practices for Spring Security?
**Answer:**
1. Never store plain text passwords
2. Use BCryptPasswordEncoder
3. Enable HTTPS in production
4. Implement CSRF protection
5. Use role-based access control
6. Log security events
7. Validate all inputs
8. Implement session timeout
