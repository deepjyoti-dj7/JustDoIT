# Operators

Operators perform operations on values and variables. JavaScript has many types of operators for different purposes.

---

## Arithmetic Operators

Perform mathematical calculations.

```javascript
let a = 10;
let b = 3;

a + b             // 13 (addition)
a - b             // 7 (subtraction)
a * b             // 30 (multiplication)
a / b             // 3.333... (division)
a % b             // 1 (modulo - remainder)
a ** b            // 1000 (exponentiation - ES2016)

// Common uses
let total = price + tax;
let remaining = budget - spent;
let area = width * height;
let average = sum / count;
let isEven = num % 2 === 0;  // Check if even
```

---

## Assignment Operators

Assign and update values.

```javascript
let x = 10;       // Basic assignment

// Compound assignment
x += 5;           // x = x + 5  → 15
x -= 3;           // x = x - 3  → 12
x *= 2;           // x = x * 2  → 24
x /= 4;           // x = x / 4  → 6
x %= 4;           // x = x % 4  → 2
x **= 3;          // x = x ** 3 → 8

// Logical assignment (ES2021)
x ||= 10;         // x = x || 10 (assign if falsy)
x &&= 20;         // x = x && 20 (assign if truthy)
x ??= 30;         // x = x ?? 30 (assign if null/undefined)
```

---

## Comparison Operators

Compare values and return boolean.

### Equality
```javascript
// Loose equality (with type coercion)
5 == "5"          // true
1 == true         // true
0 == false        // true

// Strict equality (no type coercion) ✓ Recommended
5 === "5"         // false
1 === true        // false
0 === false       // false

// Not equal
5 != "5"          // false (loose)
5 !== "5"         // true (strict) ✓ Recommended
```

### Relational
```javascript
let a = 10, b = 5;

a > b             // true (greater than)
a < b             // false (less than)
a >= 10           // true (greater than or equal)
b <= 5            // true (less than or equal)

// String comparison (lexicographical)
"apple" < "banana"     // true
"10" > "9"             // false (string comparison!)
```

---

## Logical Operators

Used for boolean logic and control flow.

### AND (&&)
Returns first falsy value or last value.

```javascript
true && true      // true
true && false     // false
false && true     // false

// Short-circuit: stops at first falsy
let result = value1 && value2 && value3;

// Practical use
if (user && user.isActive) {
  console.log("Active user");
}

// Conditional execution
isLoggedIn && redirectToDashboard();
```

### OR (||)
Returns first truthy value or last value.

```javascript
true || false     // true
false || true     // true
false || false    // false

// Default values
let name = userName || "Guest";
let config = userConfig || defaultConfig;

// Practical use
function greet(name) {
  name = name || "Friend";
  console.log("Hello, " + name);
}
```

### NOT (!)
Inverts boolean value.

```javascript
!true             // false
!false            // true
!0                // true (0 is falsy)
!""               // true (empty string is falsy)
!"hello"          // false ("hello" is truthy)

// Double NOT converts to boolean
!!"hello"         // true
!!0               // false
```

## Unary Operators

Operate on a single operand.

### Increment/Decrement
```javascript
let x = 5;

x++               // Post-increment: returns 5, then becomes 6
++x               // Pre-increment: becomes 7, then returns 7
x--               // Post-decrement: returns 7, then becomes 6
--x               // Pre-decrement: becomes 5, then returns 5

// Practical difference
let a = 5;
let b = a++;      // b = 5, a = 6 (post)

let c = 5;
let d = ++c;      // d = 6, c = 6 (pre)

// Common in loops
for (let i = 0; i < 10; i++) {
  console.log(i);
}
```

### Other Unary Operators
```javascript
+x                // Convert to number
+"42"             // 42
+true             // 1

-x                // Negate
-42               // -42
-"10"             // -10

typeof x          // Get type
typeof "hello"    // "string"

delete obj.prop   // Delete property
let obj = { a: 1 };
delete obj.a;     // true, obj is now {}

void expression   // Returns undefined
void 0            // undefined (compact way)
```

## Ternary Operator

Conditional operator - like inline if-else.

```javascript
condition ? valueIfTrue : valueIfFalse

// Basic usage
let age = 20;
let status = age >= 18 ? "adult" : "minor";

// Replace if-else
let max = a > b ? a : b;
let min = a < b ? a : b;

// Nested (use sparingly)
let grade = score >= 90 ? "A" : 
            score >= 80 ? "B" : 
            score >= 70 ? "C" : "F";

// With function calls
isValid ? save() : showError();

// Practical example
console.log(`You have ${count} item${count !== 1 ? 's' : ''}`);
```

## Nullish Coalescing (??)

Returns right side only if left is `null` or `undefined`.

```javascript
let result = value ?? "default";

// Comparison with ||
let a = 0 ?? 10;        // 0 (0 is not null/undefined)
let b = 0 || 10;        // 10 (0 is falsy)

let c = "" ?? "text";   // "" (empty string is not null/undefined)
let d = "" || "text";   // "text" (empty string is falsy)

let e = null ?? "text"; // "text" (null triggers default)
let f = undefined ?? "text"; // "text" (undefined triggers default)

// Practical use
function greet(name) {
  name = name ?? "Guest";  // Only replace null/undefined
  console.log("Hello, " + name);
}

greet("");        // "Hello, " (empty string kept)
greet(null);      // "Hello, Guest"
greet(undefined); // "Hello, Guest"
```

## Optional Chaining (?.)

Safely access nested properties without checking each level.

```javascript
// Without optional chaining
let street = user && user.address && user.address.street;

// With optional chaining
let street = user?.address?.street;

// Returns undefined if any part is null/undefined
let name = user?.profile?.name;  // undefined if user or profile is null

// With function calls
user?.getDetails?.();  // Only calls if method exists

// With arrays
let first = users?.[0];  // Safe array access
let value = obj?.['prop-name'];  // Safe dynamic property

// Practical example
function displayUserCity(user) {
  console.log(user?.address?.city ?? "City not available");
}
```

---

## Comma Operator

Evaluates multiple expressions, returns last one.

```javascript
let x = (1, 2, 3);  // x = 3

// In for loops
for (let i = 0, j = 10; i < j; i++, j--) {
  console.log(i, j);
}

// Generally avoid - reduces clarity
```

## Operator Precedence

Determines order of operations.

```javascript
// High to low (common ones):
// 1. Grouping ()
// 2. Member access . []
// 3. Function call ()
// 4. Exponentiation **
// 5. Unary + - ! typeof
// 6. Multiplication * / %
// 7. Addition + -
// 8. Comparison < > <= >=
// 9. Equality == === != !==
// 10. Logical AND &&
// 11. Logical OR ||
// 12. Nullish coalescing ??
// 13. Ternary ? :
// 14. Assignment = += -= etc.

// Examples
2 + 3 * 4         // 14 (not 20)
(2 + 3) * 4       // 20 (parentheses first)
a || b && c       // a || (b && c)
```

---

## Key Takeaways

- Use **`===` and `!==`** for comparisons (strict)
- **`&&`** returns first falsy or last value
- **`||`** returns first truthy or last value
- **`??`** only checks for null/undefined
- **`?.`** prevents errors on null/undefined access
- **Post-increment `x++`** uses then increments
- **Pre-increment `++x`** increments then uses
