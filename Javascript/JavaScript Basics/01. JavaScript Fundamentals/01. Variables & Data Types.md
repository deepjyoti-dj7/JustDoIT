# Variables & Data Types

Variables are containers for storing data values. JavaScript has three ways to declare variables and supports multiple data types.

---

## Variable Declarations

### var (Legacy - Avoid)
```javascript
var name = "John";
var age = 25;
var age = 30;  // Can redeclare (problematic)
```
- **Scope**: Function-scoped
- **Hoisting**: Declared at top, initialized as `undefined`
- **Problem**: Can be redeclared accidentally

### let (Modern - Preferred for reassignment)
```javascript
let count = 0;
count = 5;     // ✓ Can reassign
let count = 10; // ✗ Cannot redeclare
```
- **Scope**: Block-scoped `{ }`
- **Use when**: Value needs to change

### const (Modern - Default choice)
```javascript
const PI = 3.14159;
PI = 3.14;     // ✗ Error: Cannot reassign

const user = { name: "John" };
user.name = "Jane";  // ✓ Can modify object properties
user = {};           // ✗ Cannot reassign variable
```
- **Scope**: Block-scoped `{ }`
- **Use for**: Values that shouldn't be reassigned
- **Note**: Object/array contents can still change

## Data Types

JavaScript has 7 primitive types and 1 reference type.

### 1. String
Text data enclosed in quotes.

```javascript
let single = 'Hello';
let double = "World";
let template = `Hello ${name}`;  // Template literal

let multiline = `Line 1
Line 2`;  // Multi-line string
```

### 2. Number
Both integers and decimals.

```javascript
let integer = 42;
let decimal = 3.14;
let negative = -10;
let scientific = 2.5e6;  // 2,500,000

// Special values
let inf = Infinity;
let negInf = -Infinity;
let notANumber = NaN;  // Not a Number
```

### 3. Boolean
True or false values.

```javascript
let isActive = true;
let isComplete = false;

// Often used in conditions
if (isActive) {
  console.log("User is active");
}
```

### 4. Null
Intentional absence of value.

```javascript
let user = null;  // Explicitly set to "no value"
```

### 5. Undefined
Variable declared but not assigned.

```javascript
let value;
console.log(value);  // undefined

function test(param) {
  console.log(param);  // undefined if not passed
}
test();
```

### 6. Symbol (ES6)
Unique identifier.

```javascript
let id1 = Symbol("id");
let id2 = Symbol("id");
console.log(id1 === id2);  // false (always unique)
```

### 7. BigInt (ES2020)
For very large integers.

```javascript
let big = 1234567890123456789012345678901234567890n;
let alsoaBig = BigInt("9007199254740991");

// Cannot mix with regular numbers
let result = big + 10n;  // ✓ Works
let result = big + 10;   // ✗ Error
```

### 8. Object (Reference Type)
Collection of key-value pairs.

```javascript
let person = {
  name: "John",
  age: 30,
  isStudent: false
};

let array = [1, 2, 3, 4, 5];
let func = function() { return "Hello"; };
```

## Type Checking

```javascript
typeof "hello"        // "string"
typeof 42             // "number"
typeof 3.14           // "number"
typeof true           // "boolean"
typeof undefined      // "undefined"
typeof Symbol()       // "symbol"
typeof 10n            // "bigint"
typeof { }            // "object"
typeof []             // "object" ⚠️
typeof null           // "object" ⚠️ (historical bug)
typeof function(){}   // "function"
```

**Common checks:**
```javascript
Array.isArray([1, 2, 3]);     // true
value === null;                // Check for null
value === undefined;           // Check for undefined
value == null;                 // Checks both null and undefined
```

## Scope Comparison

```javascript
// var: Function scope
function testVar() {
  if (true) {
    var x = 10;
  }
  console.log(x);  // 10 (accessible outside if block)
}

// let: Block scope
function testLet() {
  if (true) {
    let y = 10;
  }
  console.log(y);  // Error: y is not defined
}

// const: Block scope
function testConst() {
  if (true) {
    const z = 10;
  }
  console.log(z);  // Error: z is not defined
}
```

## Hoisting

Variables are "hoisted" (moved to top) but initialized differently.

```javascript
// var hoisting
console.log(x);  // undefined (declared but not assigned)
var x = 5;

// Equivalent to:
var x;           // Hoisted
console.log(x);  // undefined
x = 5;

// let/const hoisting (Temporal Dead Zone)
console.log(y);  // ReferenceError: Cannot access before initialization
let y = 10;
```

---

## Best Practices

1. **Use `const` by default** - Prevents accidental reassignment
2. **Use `let` when value changes** - Loops, counters, toggles
3. **Avoid `var`** - Use in legacy code only
4. **Meaningful names** - `userName` not `x`
5. **Initialize variables** - Don't leave undefined unless intentional

```javascript
// ✓ Good
const MAX_USERS = 100;
let currentCount = 0;
const userData = { name: "John" };

// ✗ Avoid
var x;
var data;
var temp;
```

---

## Key Takeaways

- JavaScript has **7 primitive types** + **objects**
- **Primitives** are immutable (strings, numbers, booleans, etc.)
- **Objects** are mutable (can change properties)
- Use **`const`** for values that don't reassign
- Use **`let`** for values that change
- **Avoid `var`** in modern JavaScript
