## Understanding Key Files

### 1. `package.json`
Manages dependencies and scripts:
```json
{
  "name": "my-react-app",
  "version": "0.0.0",
  "scripts": {
    "dev": "vite",              // Start dev server
    "build": "vite build",      // Production build
    "preview": "vite preview"   // Preview production build
  },
  "dependencies": {
    "react": "^18.2.0",
    "react-dom": "^18.2.0"
  },
  "devDependencies": {
    "@vitejs/plugin-react": "^4.0.0",
    "vite": "^4.0.0"
  }
}
```

### 2. Entry Point (Vite: `main.jsx`)
```jsx
import React from 'react'
import ReactDOM from 'react-dom/client'
import App from './App.jsx'
import './index.css'

ReactDOM.createRoot(document.getElementById('root')).render(
  <React.StrictMode>
    <App />
  </React.StrictMode>,
)
```

### 3. Entry Point (CRA: `index.js`)
```jsx
import React from 'react';
import ReactDOM from 'react-dom/client';
import './index.css';
import App from './App';

const root = ReactDOM.createRoot(document.getElementById('root'));
root.render(
  <React.StrictMode>
    <App />
  </React.StrictMode>
);
```

### 4. Main Component (`App.jsx`)
```jsx
import { useState } from 'react'
import './App.css'

function App() {
  const [count, setCount] = useState(0)

  return (
    <div className="App">
      <h1>Hello React!</h1>
      <button onClick={() => setCount(count + 1)}>
        Count: {count}
      </button>
    </div>
  )
}

export default App
```

### 5. HTML Template (`index.html`)
```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>React App</title>
  </head>
  <body>
    <div id="root"></div>
    <script type="module" src="/src/main.jsx"></script>
  </body>
</html>
```

## Configuration Files

### Vite Configuration (`vite.config.js`)
```javascript
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [react()],
  server: {
    port: 3000,              // Custom port
    open: true               // Auto-open browser
  },
  build: {
    outDir: 'dist',          // Output directory
    sourcemap: true          // Generate sourcemaps
  }
})
```

### Environment Variables

Create `.env` file in project root:
```bash
VITE_API_URL=https://api.example.com
VITE_APP_TITLE=My React App
```

Access in code:
```jsx
const apiUrl = import.meta.env.VITE_API_URL;
const appTitle = import.meta.env.VITE_APP_TITLE;
```

**Note**: CRA uses `REACT_APP_` prefix instead of `VITE_`

## Interview Questions

### Q1: What is the difference between Vite and Create React App?
**Answer**:
- **Vite**: Modern, uses native ES modules, extremely fast HMR, smaller bundle, instant server start
- **CRA**: Traditional, uses webpack, slower builds, larger bundle, battle-tested

Vite is recommended for new projects due to superior performance.

### Q2: What is the purpose of `package.json`?
**Answer**: `package.json` is the manifest file that:
- Lists project dependencies (react, react-dom, etc.)
- Defines scripts (dev, build, test)
- Specifies project metadata (name, version, author)
- Manages development dependencies

### Q3: What is the role of `index.html` in a React app?
**Answer**: `index.html` is the single HTML page (SPA) that:
- Contains a `<div id="root">` where React mounts the app
- Includes meta tags for SEO and responsiveness
- Loads the JavaScript bundle that renders React components
- Serves as the entry point for the browser

### Q4: What is `React.StrictMode`?
**Answer**: StrictMode is a development tool that:
- Detects potential problems in the application
- Highlights unsafe lifecycle methods
- Warns about deprecated APIs
- Ensures components are pure (renders twice in dev mode)
- Only runs in development, no impact on production

```jsx
<React.StrictMode>
  <App />
</React.StrictMode>
```

### Q5: How do you add environment variables in a React app?
**Answer**:
- **Vite**: Create `.env` file, prefix with `VITE_`, access via `import.meta.env.VITE_VARIABLE`
- **CRA**: Create `.env` file, prefix with `REACT_APP_`, access via `process.env.REACT_APP_VARIABLE`

Environment variables must start with the specified prefix to be exposed to the client.

### Q6: What is the purpose of `node_modules` folder?
**Answer**: `node_modules` stores all installed dependencies (libraries and packages). It's:
- Generated when running `npm install`
- Not committed to version control (in `.gitignore`)
- Can be recreated using `package.json`
- Contains both direct and transitive dependencies
