# Pagination, Filtering & Sorting

## Why Pagination?

**Without pagination**:
- Slow response times
- High memory usage
- Poor user experience
- Database overload

**With pagination**:
- Fast, efficient queries
- Better UX (load data in chunks)
- Reduced server/DB load

> **Never return all records at once!**

---

## Page-Based Pagination

Most common and simple approach.

### Query Parameters

```
GET /api/products?page=2&limit=10
```

### Implementation

```js
app.get('/api/products', async (req, res) => {
  const page = parseInt(req.query.page) || 1;
  const limit = parseInt(req.query.limit) || 10;
  const skip = (page - 1) * limit;
  
  const products = await Product.find()
    .skip(skip)
    .limit(limit);
  
  const total = await Product.countDocuments();
  
  res.json({
    success: true,
    data: products,
    pagination: {
      page,
      limit,
      total,
      pages: Math.ceil(total / limit),
      hasNext: page < Math.ceil(total / limit),
      hasPrev: page > 1
    }
  });
});
```

### Response Format

```json
{
  "success": true,
  "data": [
    { "id": 11, "name": "Product 11" },
    { "id": 12, "name": "Product 12" }
  ],
  "pagination": {
    "page": 2,
    "limit": 10,
    "total": 100,
    "pages": 10,
    "hasNext": true,
    "hasPrev": true
  }
}
```

### Reusable Pagination Helper

```js
// utils/pagination.js
const paginate = async (model, query = {}, options = {}) => {
  const page = parseInt(options.page) || 1;
  const limit = parseInt(options.limit) || 10;
  const skip = (page - 1) * limit;
  const sort = options.sort || '-createdAt';
  
  const [data, total] = await Promise.all([
    model.find(query)
      .sort(sort)
      .skip(skip)
      .limit(limit)
      .select(options.select || '')
      .populate(options.populate || ''),
    model.countDocuments(query)
  ]);
  
  return {
    data,
    pagination: {
      page,
      limit,
      total,
      pages: Math.ceil(total / limit),
      hasNext: page < Math.ceil(total / limit),
      hasPrev: page > 1
    }
  };
};

module.exports = paginate;
```

### Usage

```js
const paginate = require('../utils/pagination');

exports.getProducts = async (req, res) => {
  const { page, limit, category } = req.query;
  
  const query = category ? { category } : {};
  
  const result = await paginate(Product, query, {
    page,
    limit,
    sort: '-createdAt',
    select: 'name price'
  });
  
  res.json({
    success: true,
    ...result
  });
};
```

## Cursor-Based Pagination

Better for real-time data and infinite scroll.

### Implementation

```js
app.get('/api/posts', async (req, res) => {
  const { cursor, limit = 10 } = req.query;
  
  const query = cursor
    ? { _id: { $lt: cursor } } // Get records before cursor
    : {};
  
  const posts = await Post.find(query)
    .sort({ _id: -1 })
    .limit(parseInt(limit));
  
  const nextCursor = posts.length > 0
    ? posts[posts.length - 1]._id
    : null;
  
  res.json({
    success: true,
    data: posts,
    nextCursor,
    hasMore: posts.length === parseInt(limit)
  });
});
```

### Usage

```
# First request
GET /api/posts?limit=10

# Next page (use nextCursor from previous response)
GET /api/posts?cursor=abc123&limit=10
```

### Response

```json
{
  "success": true,
  "data": [...],
  "nextCursor": "xyz789",
  "hasMore": true
}
```

---

## Offset-Based vs Cursor-Based

| Feature | Offset (Page) | Cursor |
|---------|---------------|--------|
| Navigation | Jump to any page | Only next/prev |
| Performance | Slower (large offsets) | Faster |
| Real-time data | Inconsistent | Consistent |
| Use case | Static data | Live feeds |
| Complexity | Simple | More complex |

**Use offset** for: Reports, admin panels, static lists
**Use cursor** for: Social feeds, infinite scroll, real-time data

## Filtering

### Basic Filtering

```
GET /api/products?category=electronics
GET /api/users?role=admin&status=active
```

```js
app.get('/api/products', async (req, res) => {
  const { category, minPrice, maxPrice, inStock } = req.query;
  
  // Build query object
  const query = {};
  
  if (category) {
    query.category = category;
  }
  
  if (minPrice || maxPrice) {
    query.price = {};
    if (minPrice) query.price.$gte = parseFloat(minPrice);
    if (maxPrice) query.price.$lte = parseFloat(maxPrice);
  }
  
  if (inStock !== undefined) {
    query.inStock = inStock === 'true';
  }
  
  const products = await Product.find(query);
  res.json({ success: true, data: products });
});
```

### Advanced Filtering (Operators)

```
GET /api/products?price[gte]=100&price[lte]=500
GET /api/users?age[gt]=18&createdAt[gte]=2024-01-01
```

```js
app.get('/api/products', async (req, res) => {
  // Remove pagination/sorting params
  const queryObj = { ...req.query };
  const excludedFields = ['page', 'sort', 'limit', 'fields'];
  excludedFields.forEach(field => delete queryObj[field]);
  
  // Convert operators (gte, gt, lte, lt)
  let queryStr = JSON.stringify(queryObj);
  queryStr = queryStr.replace(/\b(gte|gt|lte|lt)\b/g, match => `$${match}`);
  
  const query = JSON.parse(queryStr);
  const products = await Product.find(query);
  
  res.json({ success: true, data: products });
});
```

### Array Filtering

```
GET /api/products?tags=electronics,gadgets
```

```js
app.get('/api/products', async (req, res) => {
  const { tags } = req.query;
  
  const query = {};
  
  if (tags) {
    // Convert comma-separated to array
    const tagsArray = tags.split(',');
    query.tags = { $in: tagsArray };
  }
  
  const products = await Product.find(query);
  res.json({ success: true, data: products });
});
```

## Sorting

### Basic Sorting

```
GET /api/products?sort=price          # Ascending
GET /api/products?sort=-price         # Descending
GET /api/products?sort=name,-createdAt # Multiple fields
```

```js
app.get('/api/products', async (req, res) => {
  const { sort } = req.query;
  
  let sortObj = {};
  
  if (sort) {
    const fields = sort.split(',');
    fields.forEach(field => {
      if (field.startsWith('-')) {
        sortObj[field.substring(1)] = -1; // Descending
      } else {
        sortObj[field] = 1; // Ascending
      }
    });
  } else {
    sortObj = { createdAt: -1 }; // Default sort
  }
  
  const products = await Product.find().sort(sortObj);
  res.json({ success: true, data: products });
});
```

### Sorting with Pagination

```js
app.get('/api/products', async (req, res) => {
  const page = parseInt(req.query.page) || 1;
  const limit = parseInt(req.query.limit) || 10;
  const skip = (page - 1) * limit;
  const sort = req.query.sort || '-createdAt';
  
  const products = await Product.find()
    .sort(sort)
    .skip(skip)
    .limit(limit);
  
  res.json({ success: true, data: products });
});
```

## Field Selection (Sparse Fieldsets)

Return only requested fields to reduce payload size.

```
GET /api/users?fields=name,email
GET /api/products?fields=name,price,image
```

```js
app.get('/api/users', async (req, res) => {
  const { fields } = req.query;
  
  let select = '';
  if (fields) {
    select = fields.split(',').join(' ');
  }
  
  const users = await User.find().select(select);
  res.json({ success: true, data: users });
});
```

### Response

```json
[
  { "name": "John", "email": "john@example.com" },
  { "name": "Jane", "email": "jane@example.com" }
]
```

## Complete API Query Handler

### APIFeatures Class

```js
// utils/APIFeatures.js
class APIFeatures {
  constructor(query, queryString) {
    this.query = query;
    this.queryString = queryString;
  }
  
  filter() {
    const queryObj = { ...this.queryString };
    const excludedFields = ['page', 'sort', 'limit', 'fields'];
    excludedFields.forEach(field => delete queryObj[field]);
    
    let queryStr = JSON.stringify(queryObj);
    queryStr = queryStr.replace(/\b(gte|gt|lte|lt)\b/g, match => `$${match}`);
    
    this.query = this.query.find(JSON.parse(queryStr));
    return this;
  }
  
  sort() {
    if (this.queryString.sort) {
      const sortBy = this.queryString.sort.split(',').join(' ');
      this.query = this.query.sort(sortBy);
    } else {
      this.query = this.query.sort('-createdAt');
    }
    return this;
  }
  
  limitFields() {
    if (this.queryString.fields) {
      const fields = this.queryString.fields.split(',').join(' ');
      this.query = this.query.select(fields);
    } else {
      this.query = this.query.select('-__v');
    }
    return this;
  }
  
  paginate() {
    const page = parseInt(this.queryString.page) || 1;
    const limit = parseInt(this.queryString.limit) || 10;
    const skip = (page - 1) * limit;
    
    this.query = this.query.skip(skip).limit(limit);
    return this;
  }
}

module.exports = APIFeatures;
```

### Usage

```js
const APIFeatures = require('../utils/APIFeatures');

exports.getProducts = async (req, res) => {
  const features = new APIFeatures(Product.find(), req.query)
    .filter()
    .sort()
    .limitFields()
    .paginate();
  
  const products = await features.query;
  
  res.json({
    success: true,
    results: products.length,
    data: products
  });
};
```

## Best Practices

### 1. Set Default and Max Limits

```js
const page = parseInt(req.query.page) || 1;
const limit = Math.min(parseInt(req.query.limit) || 10, 100); // Max 100
```

### 2. Validate Query Parameters

```js
const { query } = require('express-validator');

router.get('/products',
  [
    query('page').optional().isInt({ min: 1 }),
    query('limit').optional().isInt({ min: 1, max: 100 }),
    query('sort').optional().isString()
  ],
  validate,
  getProducts
);
```

### 3. Index Database Fields

```js
// In Mongoose model
productSchema.index({ category: 1, price: 1 });
productSchema.index({ name: 'text', description: 'text' });
```

### 4. Cache Results

```js
const redis = require('redis');
const client = redis.createClient();

exports.getProducts = async (req, res) => {
  const cacheKey = `products:${JSON.stringify(req.query)}`;
  
  // Check cache
  const cached = await client.get(cacheKey);
  if (cached) {
    return res.json(JSON.parse(cached));
  }
  
  // Query database
  const products = await Product.find();
  
  // Cache for 5 minutes
  await client.setex(cacheKey, 300, JSON.stringify(products));
  
  res.json({ success: true, data: products });
};
```

## Interview Questions

### Q1: What's the difference between offset and cursor pagination?
**Answer**:
- **Offset**: Jump to any page, slower for large datasets
- **Cursor**: Only next/prev, faster, consistent for real-time data

### Q2: How do you handle large datasets?
**Answer**: Use pagination with reasonable limits, database indexing, and caching.

### Q3: What's a good default page size?
**Answer**: 10-50 items. Always set a maximum (e.g., 100) to prevent abuse.

### Q4: How do you sort by multiple fields?
**Answer**: `?sort=price,-createdAt` (price ascending, createdAt descending).

### Q5: Should filtering be case-sensitive?
**Answer**: Usually no. Use `{ $regex: value, $options: 'i' }` for case-insensitive search.

## Summary

- **Pagination**: Essential for large datasets
- **Page-Based**: Simple, jump to any page
- **Cursor-Based**: Better for real-time/infinite scroll
- **Filtering**: Build query from URL parameters
- **Sorting**: Use `sort` parameter with +/- prefix
- **Field Selection**: Reduce payload size
- **APIFeatures Class**: Reusable query handler
- **Best Practices**: Default limits, validation, indexing, caching
