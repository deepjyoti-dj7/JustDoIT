# 01. Introduction to Express.js

## What is Express.js?

Express is a **minimal and flexible Node.js web framework** for building web applications and APIs.

---

## Why Express?

✅ **Simple & Fast**: Minimal setup, fast development  
✅ **Routing**: Easy URL handling  
✅ **Middleware**: Modular request processing  
✅ **Template Engines**: Dynamic HTML rendering  
✅ **Large Community**: Extensive packages and support  

---

## Features

- **Routing**: Handle different HTTP methods (GET, POST, PUT, DELETE)
- **Middleware**: Process requests (auth, logging, parsing)
- **Template Engines**: EJS, Pug, Handlebars
- **Static Files**: Serve CSS, JS, images
- **Error Handling**: Centralized error management
- **RESTful APIs**: Easy API development

---

## Express vs Node.js

| Feature | Node.js | Express |
|---------|---------|----------|
| **Type** | Runtime | Framework |
| **Routing** | Manual | Built-in |
| **Middleware** | No | Yes |
| **Code** | Verbose | Concise |
| **Setup** | Complex | Simple |

---

## When to Use Express?

✅ Building REST APIs  
✅ Web applications  
✅ Real-time apps (with Socket.io)  
✅ Microservices  
✅ Server-side rendering  

---

## Popular Uses

- **APIs**: Backend for React/Vue/Angular apps
- **Full-Stack Apps**: With EJS/Pug templates
- **Microservices**: Lightweight services
- **Proxies**: API gateway

---

## Summary

**Express.js** = Fast, minimalist web framework for Node.js  
**Purpose**: Build web apps and APIs quickly  
**Key Features**: Routing, Middleware, Templates, Error Handling
