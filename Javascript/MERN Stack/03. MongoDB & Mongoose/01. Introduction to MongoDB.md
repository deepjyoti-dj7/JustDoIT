# 01. Introduction to MongoDB

## What is MongoDB?

MongoDB is a **NoSQL document database** that stores data in flexible, JSON-like documents.

---

## Why MongoDB?

✅ **Flexible Schema**: No fixed structure  
✅ **Scalable**: Horizontal scaling (sharding)  
✅ **Fast**: High performance  
✅ **JSON-like**: Easy to work with JavaScript  
✅ **Rich Queries**: Powerful query language  

---

## Key Features

- **Document-Oriented**: Stores data as documents (BSON)
- **No Schema**: Flexible data structure
- **Indexing**: Fast data retrieval
- **Aggregation**: Data processing pipelines
- **Replication**: High availability
- **Sharding**: Horizontal scaling

---

## SQL vs MongoDB

| SQL | MongoDB |
|-----|----------|
| Database | Database |
| Table | Collection |
| Row | Document |
| Column | Field |
| JOIN | Embedding/Referencing |
| Fixed Schema | Flexible Schema |

**SQL:**
```sql
SELECT * FROM users WHERE age > 25;
```

**MongoDB:**
```javascript
db.users.find({ age: { $gt: 25 } });
```

---

## Document Structure

**JSON-like format (BSON):**

```json
{
  "_id": "507f1f77bcf86cd799439011",
  "name": "John Doe",
  "age": 30,
  "email": "john@example.com",
  "address": {
    "city": "New York",
    "country": "USA"
  },
  "hobbies": ["reading", "gaming"]
}
```

---

## When to Use MongoDB?

✅ **Flexible schema**: Data structure changes frequently  
✅ **Rapid development**: Fast prototyping  
✅ **Big data**: Large volumes of data  
✅ **Real-time analytics**: Fast queries  
✅ **Content management**: Blogs, catalogs  

❌ **Not for**: Complex transactions, strict ACID requirements

---

## MongoDB Components

- **mongod**: Database server
- **mongos**: Query router (sharding)
- **mongo**: Shell client
- **MongoDB Compass**: GUI tool
- **MongoDB Atlas**: Cloud database

---

## Summary

**MongoDB** = NoSQL document database  
**Stores**: JSON-like documents (BSON)  
**Benefits**: Flexible schema, scalable, fast  
**Use Cases**: Apps with evolving data, real-time analytics
